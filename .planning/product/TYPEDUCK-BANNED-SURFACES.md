# TypeDuck Banned Surfaces Contract

**Capture date:** 2026-06-23  
**Decision traceability:** D-12, D-13, D-14, D-15, D-16, D-17  
**Related requirements:** IDEN-04, LANG-02, SEC-02, SEC-03, VER-02

## Purpose

This negative contract defines scaffold surfaces that are banned from the first TypeDuck Windows IME release. Phase 1 documents the rules and evidence only. Per D-15 and D-16, cleanup, production renames, protocol removal, installer changes, and automated user-facing resource checks belong to later mapped phases.

Per D-14, each row includes exact audit patterns and likely files so later phases can verify removal without guessing. Phase 6 owns broad cleanup and automated leak checks unless another cleanup phase is listed.

## Phase Boundary

- D-12 creates this negative contract.
- D-13 defines the minimum banned categories: visible Moqi branding, fcitx references, WebDAV/cloud clipboard, AI controls, Simplified-only Chinese copy, arbitrary backend-declared config tools, excessive customization beyond Web alpha, and legacy Moqi backend fallback behavior.
- D-14 requires audit patterns and likely files for later verification.
- D-15 and D-16 keep Phase 1 documentation-only; this file must not trigger production code, installer, registry, protocol, runtime path, or workflow edits in this plan.
- D-17 limits Phase 1 requirement completion to IDEN-02 and VER-01. This artifact supports later IDEN-04, LANG-02, SEC-02, SEC-03, and VER-02 work but does not complete those requirements by itself.

## Later Verification Notes

- Treat matches in tests, historical docs, or migration notes separately from user-facing runtime surfaces, but require an explicit allowlist for any remaining Moqi/fcitx/WebDAV/cloud/AI/Simplified-only text.
- For user-facing copy, verify both source strings and compiled resources where applicable.
- For privacy/security surfaces, removal must include protocol fields, launcher listeners, backend dispatch, docs, settings entry points, and generated release artifacts.
- Automated user-facing resource checks should live in a later verification plan, likely Phase 6 for VER-02.
