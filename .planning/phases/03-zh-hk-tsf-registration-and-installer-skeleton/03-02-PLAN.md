<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Installer elevation -> system directories | SetupHelper copies and registers TSF DLLs into Windows system directories as admin. |
| Installer script -> user startup state | Inno writes HKCU startup entries and launches the TypeDuck launcher. |
| Uninstall cleanup -> registry/system files | Cleanup removes COM/TSF/startup/task/system-DLL state and must not delete unrelated data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Elevation of Privilege | SetupHelper source DLL paths | mitigate | Copy only the TypeDuck DLL paths under the installer app directory and keep quoted command lines. |
| T-03-02-02 | Tampering | Dual-bitness DLL deployment | mitigate | Static check and VM plan verify Win32 SysWOW64 and x64 System32 registration paths stay synchronized. |
| T-03-02-03 | Spoofing | Startup registry value | mitigate | Use TypeDuckLauncher value/path and avoid legacy Moqi launcher startup keys except narrow cleanup. |
| T-03-02-04 | Denial of Service | Locked DLL replacement/reboot fallback | mitigate | Preserve scheduled re-registration mechanics under TypeDuckIME-ReRegisterTSF and verify cleanup. |
| T-03-02-05 | Tampering | Legacy migration cleanup | mitigate | Keep Moqi cleanup allowlisted to known scaffold CLSID/files only; do not broad-delete user data or unrelated Moqi installations. |
</threat_model>
