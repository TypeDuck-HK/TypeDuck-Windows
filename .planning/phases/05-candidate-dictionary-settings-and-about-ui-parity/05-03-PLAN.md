<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Settings UI to launcher/backend | User-controlled settings values cross into persistence and engine customization. |
| JSON preferences to Rime customization | Local JSON preferences generate YAML/custom-settings side effects consumed by the engine. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03-01 | Tampering | preferences JSON | mitigate | Validate schema, ranges, known language IDs, and default fallback before applying settings. |
| T-05-03-02 | Denial of Service | deploy/reconfigure path | mitigate | Batch changes behind Apply, bound deploy failure state, and avoid repeated deploys for each control toggle per D-19 and D-47. |
| T-05-03-03 | Repudiation | settings apply result | mitigate | Return explicit success/failure status for settings apply while avoiding raw typed content in logs. |
| T-05-03-SC | Tampering | package installs | accept | No npm, pip, or cargo installs are planned. |
</threat_model>
