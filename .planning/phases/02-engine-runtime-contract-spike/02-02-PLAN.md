## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Probe process -> adapter process | Developer-owned probe sends framed protobuf to a backend process over stdio. |
| Adapter process -> TypeDuck runtime files | Backend dynamically loads staged rime.dll and schema/plugin assets. |
| External sibling checkout -> repo-local patch evidence | Any required moqi-ime adapter delta may originate outside this repo but must be represented as repo-local patch evidence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-02-02-02 | Tampering | adapter runtime overlay | mitigate | Overlay only into an isolated proof working directory and record the staged runtime manifest hash references. |
| T-02-02-03 | Denial of Service | TypeduckBackendProbe frame reads | mitigate | Add bounded read timeouts and maximum frame sizes in the probe even though production protocol hardening belongs to Phase 4. |
| T-02-02-04 | Repudiation | typing-proof.json | mitigate | Record probe version, input sequence, adapter path/hash, runtime manifest path/hash, and command exit codes. |
| T-02-02-05 | Information Disclosure | backend-stdio-frames.ndjson | mitigate | Use only controlled sample inputs from fixtures and do not capture personal user typing. |
