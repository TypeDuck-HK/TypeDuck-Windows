<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Proof harness to backend/runtime | Harness launches or talks to local TypeDuck runtime artifacts and must not validate legacy fallback as product behavior. |
| Windows VM to host | Smoke verification may install/register TSF components and must run in a disposable VM by default. |
| Evidence files to planner/verifier | JSON/checklist evidence can be incomplete or fabricated unless validated against commands and required cases. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04-01 | Repudiation | `typing-mvp-evidence.json` | mitigate | Validator requires commands, timestamps, artifact paths, and pass/fail for every golden case. |
| T-04-04-02 | Tampering | runtime proof source | mitigate | Reuse Phase 2 staged runtime and reject legacy fallback-only evidence per Phase 2 D-14/D-15. |
| T-04-04-03 | Denial of Service | Windows smoke | mitigate | Exercise timeout/restart behavior in a disposable VM and require bounded host-app behavior evidence. |
| T-04-04-04 | Information Disclosure | typing evidence | mitigate | Store only controlled fixture input/output, not arbitrary user text, and keep raw lookup evidence limited to fixture candidates. |
| T-04-04-SC | Tampering | package installs | accept | No npm, pip, or cargo installs are planned. |
</threat_model>
