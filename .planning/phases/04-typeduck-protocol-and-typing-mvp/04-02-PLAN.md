<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TSF client to launcher | Framed client requests arrive over a same-user named pipe and may be malformed or oversized. |
| Launcher to backend process | Backend process may fail to start, hang, crash, restart, or emit malformed stdout frames. |
| Backend manifest to launcher | Manifest text controls local backend command selection and must not become product authority for TSF identity. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-04-02-01 | Denial of Service | `PipeClient` timeout path | mitigate | Return a bounded TypeDuck error/degraded response and stop timers before restart per D-14 and TYPE-05. |
| T-04-02-02 | Tampering | backend stdout frames | mitigate | Use bounded `FrameBuffer` and reject malformed responses before client routing per D-13. |
| T-04-02-03 | Spoofing | backend profile mapping | mitigate | Seed mapping from first-party TypeDuck profile GUID and keep backend metadata optional per Phase 3 D-01/D-04. |
| T-04-02-04 | Repudiation | recovery evidence | mitigate | Write `launcher-recovery.json` fixture outcomes and summary evidence for missing/restart/timeout cases. |
| T-04-02-SC | Tampering | package installs | accept | No npm, pip, or cargo installs are planned. |
</threat_model>
