|----------|-------------|
| Backend response to TSF client | Candidate, commit, health, and error data crosses from backend/launcher into host-process TSF code. |
| TSF key event to backend | User key events are forwarded through synchronous frontend code and must not hang host apps. |
| Raw candidate comment to candidate UI | Untrusted raw lookup-filter comments become visible text in the existing comment position. |
